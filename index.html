Laws of UX Considered and used (as planned in the preliminary research discussion)

  • Aesthtic usability effect

    To some extent I considered this law in terms of the typography and colour of the website. Firstly before I chose my fonts, I did research as to what font is best used for a security-based website and it came up with bold sans-serif fonts so I searched that type of font and came across Hemi-Head and Quicksand in which the Hemi-head I chose for the logo and the Quicksand for the text and I thought these complimented each other well. I also did some testing in terms of the shade of blue from my logo which would go well with the text so that the relationship between the font and colour could create a more aesthetically pleasing display which would enhance readability for users.

  • Doherty Threshold

    I definitely considered this law with the amount of images in particular that I have on my website and this could potentially have slowed my website down significantly which in turn would make users frustrated and make them leave the website which would demean the purpose of it. To use this law, I took all of my images into photoshop before putting it into my website and I reduced the file size by reducing the resolution and changing all images to a JPEG file name which made the images load faster on the websites, saving time for users.

  • Hicks Law

    In some retrospect, I considered and implemented Hicks Law, particularly in the News and Awareness page with the section box full of external news articles. Instead of putting only a few articles, I decided to add more as I thought this page would otherwise lack content and with this, I decided to put all articles in one section for a more interesting presentability which In turn may tend to increase time that users spend on the website as their choice of which article to click on may be extended (in a good way) – as there are so many options in one section.

  • Jakob’s Law

    In terms of the layout of my website (from header to footer), I used a fairly simple layout theme that most websites would either use of base their preliminary one on and make a few adjustments and in some degree, I did consider this throughout every page in my website so that usability can be enhanced and it’s a more efficient process for users to navigate around the website.

  • Law of Common Region

    When coding my website, I tried to use the flexbox attribute in as many places as I can as this allows me to group certain elements together whilst still keeping the page presentable and readable for users. – Using flexbox was also easier for me to code instead of fiddling and wasting time trying to position them in the right place which would be more of a trial and error situation. Doing this, had helped me to implement this law which would have a positive impact on users when wanting to find specific sections of information which would have been negatively impacted if information wasn’t grouped together.

  • Law of proximity

    Relating to the above law – The flexbox along with the creation of certain divs helped me to implement this law and I very strongly considered this along with alignment right from my block framing stages as I wanted to make sure that elements have a relationship with each other on the pages and the header to footer all have a flow to the design so when users a looking from top to bottom of the page, they are not confused. I also made sure elements (for example on the first page) that different elements were still close enough to each other which keeping things proxime and aligned allowed the usability and readability towards uses to be increased.

  • Law of similarity

    The implementation of this law was apparent right from my wireframing phase and I was already planning on keeping the consistency of typography, colour, and layout theme of each page almost the same so users could have an easy experience using my website. Along with this, I also tried to keep the amount of page content similarly low on each page so that users aren’t going through lengthy process in terms of the overload of reading content and doing this similar style for each page enhances the user-website relationship.
